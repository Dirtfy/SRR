**W1 — `StorageUploadTest` creates real files in `context.cacheDir`**

Tests create small bitmap JPEGs in the app cache dir and clean up with `file.delete()` in the happy path. If a test fails before `file.delete()`, the file persists in the cache. Acceptable for a prototype — cache is not committed to git, and files are tiny.

**W2 — Storage emulator binary not cached; requires download on first run**

`firebase setup:emulators:storage` must be run once before `firebase emulators:start` includes Storage (port 9199). Added to known setup steps. No code change needed.

**W3 — `uploadItemImage_unauthenticated_fails` signs out then does not re-sign-in**

The `@After tearDown` still calls the auth emulator reset endpoint, which is safe since there is no current user. No issue.
