# Debat Log — 2026-06-27e

**Role:** Debator  
**Scope:** Four user-reported issues

---

## Issue 1 — Wrong tab after adding a feature

`loadAllData()` captures no previous state before setting `UserUiState.Loading`. The new `Ready` is always constructed with `activeTab = Tab.ITEMS` (the default), discarding whatever tab was active. Fix: capture `activeTab` from the current `Ready` state before emitting `Loading`, then restore it in the new `Ready`.

**Severity:** CRITICAL (user-visible regression every time a feature is added)

---

## Issue 2 — Instrumented tests do not cover evaluation threshold

`RemoteEvaluationRepositoryTest` tests repository read/write. No test exercises the full path: 3 users submit → scoring engine runs → score becomes non-null. Fix: add `threeUsersEvaluate_scoreExceedsThreshold` test that creates 3 accounts, each submits an evaluation for the same feature, then runs `DefaultFeatureScoringEngine` inline and asserts the result is non-null (threshold met).

**Severity:** WARNING (coverage gap, not a runtime bug)

---

## Issue 3 — Logout immediately re-logins (debugAutoLogin undoes sign-out)

`signOut()` calls `Firebase.auth.signOut()` then replaces with a fresh `LoginFragment`. That fragment's `LaunchedEffect(Unit)` calls `debugAutoLogin()` since `isAlreadySignedIn()` is now false — immediately signing back in. User can never actually log out in a debug build.

Fix: pass `autoLogin: Boolean` as a Fragment argument. `LoginFragment.newInstance(autoLogin = false)` is used by `signOut()`; default (`true`) is used on cold start.

**Severity:** CRITICAL

---

## Issue 4 — Creator tracking, delete, UI hint

**4a — createdBy field missing**  
`Item` and `Feature` have no `createdBy`. Firestore documents written before this change also have no field. Fix: add `createdBy: String = ""` with default for backward compat. Read and write it in remote repos.

**4b — Delete not allowed in security rules**  
Current rules have `create` but no `delete`. Fix: add `allow delete: if request.auth != null && resource.data.createdBy == request.auth.uid`.

**4c — No delete in repository/ViewModel/UI**  
Add `deleteItem` / `deleteFeature` to repository interfaces and remote implementations. Add `onDeleteItem` / `onDeleteFeature` to `UserViewModel`. Show a trailing delete `IconButton` in the list only when `item.createdBy == currentUserId`.

**4d — No creator UI hint**  
Show "Added by you" as `supportingContent` in `ListItem` for owned entries.

**Severity:** CRITICAL (no way to delete, which is a promised feature)

### MINOR

- No delete confirmation dialog (accidental deletion risk). Acceptable for now — tap is targeted and list reloads immediately.

Two CRITICALs (tab reset, logout), one WARNING (test gap), one CRITICAL cluster (creator/delete). Developer fixes all before commit.
