## Findings

### CRITICAL — None

### WARNING

**W1: `imageUrl` is not validated (length, scheme)**  
The URL is stored verbatim in Firestore and loaded by Coil. An invalid/unreachable URL will simply render nothing (Coil fails silently). For a prototype this is acceptable, but long URLs could bloat documents.  
*Decision: acceptable for prototype. No Firestore rule change needed.*

**W2: No INTERNET permission audit**  
Coil fetches over HTTP/HTTPS. The manifest must have `<uses-permission android:name="android.permission.INTERNET" />`. Android Studio projects created with Firebase already include it, but not verified here.

### MINOR

**M1: No loading/error placeholder in `AsyncImage` calls**  
Both `Popup.kt` and `Screen.kt` use `AsyncImage` without `placeholder` or `error` painters. The placeholder box in the popup covers the no-URL case but not the slow-load case (blank space during fetch).  
*Low priority for prototype.*

**M2: `gradle.properties` memory increase changes build environment**  
Heap raised to 2560 m. This may be higher than needed on low-RAM machines but is necessary with Kotlin 2.0.21 + Compose + Coil. The old OOM was intermittent; the new setting is conservative.
