## Findings

### CRITICAL — Emulator host `10.0.2.2` unreachable on physical device

**File:** `app/src/main/java/com/dirtfy/srr/SRRApplication.kt`  
**Severity:** CRITICAL

`10.0.2.2` is the magic AVD loopback address that routes to the host machine. On a physical Android device it is a regular non-routable IP; the Firebase emulator is unreachable. The correct host for physical devices is `localhost` combined with `adb reverse tcp:9099 tcp:9099 && adb reverse tcp:8080 tcp:8080`. `adb reverse` also works on AVDs, so `"localhost"` is the universal host for both device types when ADB reverse is set up.

**Fix applied:** `"10.0.2.2"` → `"localhost"` in both `useEmulator` calls.
