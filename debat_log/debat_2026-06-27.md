### MINOR ŌĆö Duplicate `useEmulator()` calls: `SRRApplication.onCreate()` + test `@BeforeClass`

**Files:** `SRRApplication.kt`, `RemoteEvaluationRepositoryTest.kt`

`SRRApplication.onCreate()` (which runs before any instrumented test) already calls:
```kotlin
Firebase.auth.useEmulator("localhost", 9099)
Firebase.firestore.useEmulator("localhost", 8080)
```

Then `RemoteEvaluationRepositoryTest.setUpEmulators()` (`@BeforeClass`) calls the same methods again with identical arguments. The Firebase SDK has historically tolerated a second identical `useEmulator()` call on the same port, but it is officially documented as a one-shot call.

**Impact:** Tests passed in prior sessions, so the SDK appears tolerant. However, a future Firebase BOM update could make this throw `IllegalStateException`.  
**Mitigation path:** Guard with `try/catch` in `@BeforeClass`, or suppress the second call by checking if already configured (no public API for this currently).  
**Decision:** MINOR ŌĆö does not block the current run.
