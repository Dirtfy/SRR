## Design Reviewed

- User taps Add Item → dialog with name + image picker area
- Tapping the image area opens Android Photo Picker (`PickVisualMedia`)
- URI returned → upload to `Firebase.storage.reference("items/{uuid}.jpg")` starts immediately (background)
- `AddItemDialogState` holds: `imageUri: Uri?` (local preview), `imageUrl: String?` (Storage URL after upload), `isUploadingImage: Boolean`
- While uploading: thumbnail shows with semi-transparent overlay + spinner; Add button disabled
- On Add: `createItem(name, createdBy, imageUrl = dialog.imageUrl)` — same Firestore write as before
- Detail page + map popup: unchanged — already use Coil `AsyncImage`

## Findings

### CRITICAL — None

### WARNING

**W1: Debug builds upload to REAL Firebase Storage (not emulator)**  
The Firebase Storage emulator (port 9199) is not wired up. Debug builds write item images to the production Storage bucket. This means:
- Dev/test image uploads are billed (small cost for prototype)
- Images uploaded during development persist in production unless manually cleaned  
*Decision: acceptable for prototype. Storage emulator setup is disproportionate effort. Noted here for future work.*

**W2: No Storage cleanup on item delete**  
`deleteItem(id)` removes the Firestore document but leaves the image in Storage. Orphaned blobs accumulate over time.  
*Decision: acceptable for prototype. Add cleanup task to README.*

**W3: Upload-then-dismiss race — orphaned Storage object**  
If the user picks an image, upload starts, then user dismisses the dialog before upload completes: the coroutine finishes after dismiss, uploads the file to Storage, but the URL is discarded (dialog state is null). The Storage object is never referenced.  
*Same root cause as W2. Acceptable for prototype.*

### MINOR

**M1: No client-side image size / type validation**  
`PickVisualMedia.ImageOnly` enforces image MIME type at the picker level, but not file size. A 50 MB photo would upload successfully (slowly).

**M2: No image compression before upload**  
Full-resolution photos from modern phones can be 5–15 MB. No resize or quality reduction is applied.
