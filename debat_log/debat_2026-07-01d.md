## Changes Reviewed

### Bug: old Storage image not deleted after item-image update

Root cause: `RemoteStorageRepository.deleteImage(url)` used `Firebase.storage.getReferenceFromUrl(url)`
with a production download URL (`https://firebasestorage.googleapis.com/v0/b/.../o/items%2F…?alt=media&token=…`).
The Firebase SDK does not reliably parse the `?alt=media&token=…` query tail in all SDK versions,
causing a silent failure wrapped inside `runCatching`. The `Result.failure` was discarded at the
call site in `UserViewModel.onSubmitEditItemImage`.

Fix:
- Added `core/util/StorageUrlUtil.kt` — pure JVM `extractStoragePath(url)` function that finds `/o/`
  in the URL and `URLDecoder.decode`s the path segment (e.g. `items/uuid.jpg`).
- `RemoteStorageRepository.deleteImage` now calls `Firebase.storage.reference.child(path).delete()`
  which is independent of URL-token parsing. Falls back to `getReferenceFromUrl` if extraction fails.

### Feature: full-size image preview popup on item detail

- `UserUiState.Ready.imagePreviewUrl: String?` — null means no popup; non-null triggers dialog.
- `UserViewModel.onShowItemImagePreview(url)` / `onDismissItemImagePreview()`.
- `ImagePreviewDialog` composable: full-screen `Dialog` with `usePlatformDefaultWidth=false`,
  black 90 % scrim, `AsyncImage` at `ContentScale.Fit` filling the screen. Dismiss on tap.
- Image `Box` in `UserItemDetailContent` gains `Modifier.clickable { onPreviewImage(url) }` when
  a URL is present. Edit button (`IconButton`) is inside the Box and captures its own click —
  it does NOT trigger the preview.
- Callbacks threaded: `UserScreen` → `UserReadyContent` → `UserItemDetailContent`; dialog overlay
  rendered at `UserScreen` level alongside other dialogs.
- `UserFragment` wired for both new callbacks.
