## Findings

**W1 — 'S' stroke path is hand-crafted, not derived from a typeface**

The three-segment bezier path approximates a bold S but is not pixel-perfect
compared to a real font glyph. Acceptable for a prototype icon — consistent
with the overall placeholder-asset approach of the project.

**W2 — debug/res/drawable only overrides background; foreground is shared**

The white S foreground vector lives in src/main. If a debug-only foreground
variation were needed (e.g., "D" watermark), src/debug/res/drawable/
ic_launcher_foreground.xml would need to be added. Not needed now.

**M1 — Python script left in scratchpad, not in project**

The gen_icons.py script is in a session temp directory. Not a build artifact,
no automated icon regeneration. Re-run manually if densities change. Low
impact for a prototype.
