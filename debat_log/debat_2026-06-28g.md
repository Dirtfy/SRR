## Findings

**W1 — MIN_EVALUATORS constant duplicated across files**

`MIN_EVALUATORS = 3` (user/Screen.kt) and `SCORE_THRESHOLD = 3` (compilation/Screen.kt) are both
hardcoded to match `LoadFeatureScoresUseCase.minVoteThreshold = 3`. They will drift if the
threshold ever changes. Acceptable for prototype — a single source of truth would require
passing the threshold through the state, adding unnecessary complexity now.

**W2 — Map hint is always visible even before features are selected**

When X/Y feature dropdowns show empty strings, the hint still says "Only items with 3+ evaluations…(0 of N)". This is harmless since the map itself shows "Please select features for both axes" — the hint doesn't contradict, just appears slightly early.
