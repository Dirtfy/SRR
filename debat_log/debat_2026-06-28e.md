## CRITICAL — Root cause of upload permission denied

**C1 — Auth emulator token rejected by production Firebase Storage**

`SRRApplication.kt` wired Auth and Firestore to the emulator but left Storage
pointing at production Firebase. Emulator-issued auth tokens are not recognized
by production services — `request.auth` evaluates to null on the production
Storage side, so even the base `request.auth != null` rule rejected every write.

**Fix:**
- `SRRApplication.kt`: added `Firebase.storage.useEmulator("localhost", 9199)`
  inside the `USE_EMULATOR` block, alongside the existing Auth and Firestore lines.
- `firebase.json`: added `"storage": {"port": 9199}` to the `emulators` section
  so `firebase emulators:start` launches the Storage emulator alongside Auth and Firestore.
- User must restart Firebase emulators and run `adb reverse tcp:9199 tcp:9199`
  on a physical device before retesting.
