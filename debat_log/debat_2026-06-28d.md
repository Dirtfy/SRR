**Scope:** Storage upload permission bug fix

## Bug Found (CRITICAL)

**C1 — Storage write rejected: permission denied on image upload**

Root cause: `storage.rules` had `request.resource.contentType.matches('image/.*')` as a write condition.
When `putFile(uri)` is called without explicit metadata, the Firebase Storage SDK may not
auto-detect the MIME type for Android `content://` URIs, resulting in a null or `application/octet-stream`
content type that fails the regex check — producing "user does not have permission to access this object".

**Fixes applied:**
1. `storage.rules` — removed `contentType.matches(...)` condition; kept auth + 10 MB size guard.
   Client-side `PickVisualMedia.ImageOnly` already enforces image-only selection.
2. `RemoteStorageRepository` — now accepts `Context` (passed from `Application`) and resolves
   the real MIME type via `ContentResolver.getType(uri)` before upload, setting it explicitly
   in `StorageMetadata`. Falls back to `"image/jpeg"` if resolver returns null.
3. `UserViewModel.factory()` — signature changed to `factory(application: Application)` so
   context is available when constructing `RemoteStorageRepository`.
4. `UserFragment` — calls `UserViewModel.factory(requireActivity().application)`.
5. `storage.rules` redeployed to production.
