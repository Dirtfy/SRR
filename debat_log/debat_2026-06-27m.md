## Changes Reviewed

### 1. `SRRApplication.kt` — `setPersistenceEnabled(false)` before `useEmulator()`

| Severity | Finding |
|----------|---------|
| ✅ CORRECT | `setPersistenceEnabled(false)` is set BEFORE `useEmulator()`, which is required because both settings need the Firestore client to be uninitialized. Correct ordering confirmed. |
| ✅ CORRECT | `BuildConfig.USE_EMULATOR` guard means production builds (release/staging) are unaffected. |

### 2. `RemoteItemRepository.kt` — `get(Source.SERVER)` in `createItem()` and `getAllItems()`

| Severity | Finding |
|----------|---------|
| ✅ CORRECT | `Source.SERVER` bypasses the local cache. With `setPersistenceEnabled(false)`, writes complete only after server ack, so the `Source.SERVER` read immediately sees just-written documents. |
| MINOR | The `getAllItems()` endpoint scans the full collection on every `createItem()` call (for uniqueness). Acceptable for a prototype; would need an index-based query in production. |

### 3. `RemoteFeatureRepository.kt` — same pattern as `RemoteItemRepository.kt`

| Severity | Finding |
|----------|---------|
| ✅ CORRECT | Consistent with `RemoteItemRepository` implementation. |

### 4. `deleteUrl()` in all three test files — missing `responseCode` read

| Severity | Finding |
|----------|---------|
| **CRITICAL (FIXED)** | `HttpURLConnection` in Android's OkHttp implementation does not actually send the HTTP request until the response is triggered. Calling `connect()` then `disconnect()` without reading `responseCode` leaves the DELETE unsent. Documents persisted between tests causing 13 failures. Fixed by replacing `connect()` with `conn.responseCode`. |

### 5. UI test "No compose hierarchies" — 15/16 `UserScreenUiTest` failures

| Severity | Finding |
|----------|---------|
| **RESOLVED (stale build artifact)** | Failures reproduced after a clean build of the previous session's changes without clearing. A `./gradlew clean` eliminated all UI test failures. Root cause: stale compiled classes from a previous incremental build that did not reflect the full UI change set. No code changes required. |
