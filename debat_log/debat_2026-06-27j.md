## Findings

### CRITICAL

### WARNING

**W1 — Thread.sleep(150) in ordering tests**
`items_returnedInCreationOrder` and `features_returnedInCreationOrder` use `Thread.sleep(150)` to
force distinct Firestore `createdAt` server timestamps. The Firestore emulator is local so 150 ms is
ample, but on a slow CI host it could theoretically fail. No clean alternative without adding a
client-side clock mock — accepted for now.

**W2 — `featureDetail_itemsSortedByScore_highestFirst` uses `positionInRoot.y`**
This approach relies on the fact that items are rendered in a single-column scrollable list. If the
layout ever switches to a Grid this test would need updating. This is the correct approach for a
LazyColumn — no better alternative without semantic ordering tags.

**W3 — `loadingState_showsProgressIndicator` uses contentDescription**
`CircularProgressIndicator` does not have a fixed contentDescription — M3's implementation may
differ across Compose versions. Should be verified on the actual device run. Fallback: assert by
tag if the CD is not stable.

### MINOR

**M1 — emulator useEmulator() double-call guard**
Both new test files use try-catch to guard against duplicate `useEmulator()` calls when multiple
test classes share a JVM process. This is correct; the same pattern is used in
`RemoteEvaluationRepositoryTest`.

device. Ready to commit pending instrumented test pass.
