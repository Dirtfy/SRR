## Findings

### CRITICAL — No Firebase emulator bootstrap in the app process

**File:** (missing) `SRRApplication.kt` + `AndroidManifest.xml`  
**Severity:** CRITICAL

The Firebase Auth emulator is configured only inside `RemoteEvaluationRepositoryTest.setUpEmulators()` (`@BeforeClass`). This redirect applies exclusively during instrumented test runs. When the app runs normally (debug build on device/AVD), `Firebase.auth` connects to production Firebase at project `shared-relative-rank`.

Production Firebase returns `CONFIGURATION_NOT_FOUND` because the Email/Password sign-in provider is not enabled in the Firebase Console for that project. There is no `android:name` on the `<application>` tag in `AndroidManifest.xml`, so no `Application.onCreate()` hook exists to call `Firebase.auth.useEmulator()` before any Fragment or ViewModel touches `Firebase.auth`.

**Fix required:**
- Create `SRRApplication.kt` with `Firebase.auth.useEmulator("10.0.2.2", 9099)` and `Firebase.firestore.useEmulator("10.0.2.2", 8080)` gated on `BuildConfig.DEBUG`.
- Add `buildConfig = true` under `buildFeatures` in `app/build.gradle.kts` (AGP 8.x does not generate `BuildConfig` by default).
- Register `android:name=".SRRApplication"` in `AndroidManifest.xml`.

### WARNING — Test emulator host is `localhost`, not `10.0.2.2`

**File:** `RemoteEvaluationRepositoryTest.kt:41`  
**Severity:** WARNING

`setUpEmulators()` hard-codes `"localhost"` for both Auth and Firestore. On an AVD, `localhost` resolves to the virtual device itself, not the host machine where `firebase emulators:start` is running. The tests only passed on the previous session because ADB reverse forwarding (`adb reverse tcp:9099 tcp:9099`) mapped AVD-`localhost:9099` to host-`localhost:9099`. Without ADB reverse, AVD instrumented tests will silently connect to nothing and hang.

**Recommended fix (non-blocking for this commit):** Extract a constant `EMULATOR_HOST` defaulting to `"10.0.2.2"` for AVD or drive it via a `BuildConfig` field. For now, the ADB reverse workaround documented in RULEBOOK is sufficient.
