### MINOR

**M1 — `loadAllData` optional param silently affects all callers**
File: `UserViewModel.kt`, `loadAllData`

All other `loadAllData()` calls (post-delete, post-evaluate, post-add) now clear `selectedItem` because they pass no arg (default null). This is the correct behavior (clearing a deleted item's selection), but a future caller passing a wrong `selectedItemId` could accidentally keep a stale item visible. Acceptable for the current single-module scope.

## Root-cause Analysis (Instrumented Tests)

Investigated why `connectedDebugAndroidTest` failed with "Process crashed" after ~37 tests:

1. **Samsung Backup** kills the `com.dirtfy.srr` process during test runs longer than ~45 s (`bkup TRNB` death reason in logcat).
2. **Fix applied:** `android:allowBackup="false"` in `AndroidManifest.xml` and `adb shell settings put global backup_enabled 0` reduce (but do not fully eliminate) interference.
3. **Workaround adopted:** Run each instrumented test class as a separate Gradle invocation (≤ 50 s each). All 54 instrumented tests pass this way.
