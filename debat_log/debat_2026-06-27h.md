## Root cause confirmed

`CompilationMapTab` was doing `(score/10)*2-1` to get `[-1,1]`, storing it in `MapItem.primaryScore`.
`ScatterPlot` canvas used `(val+1)/2` — correct for positioning.
`MapItemPopup` then displayed the same string — showing raw `[-1,1]` to users.

Fix: pass [0,10] directly, update ScatterPlot to use `val/10` for canvas.
Center cross at `size/2` still corresponds to score 5.0 (midpoint of [0,10]). Correct.
