## Findings

### WARNING

**W1 — Old Firestore documents lacking `createdAt` are excluded by `orderBy`**
Firestore excludes documents from a query that don't have the indexed field when `orderBy` is used. Any items/features added before this change (missing `createdAt`) will disappear from the list until they're re-added.
Mitigation: The Firebase emulator is cleared before every instrumented test run, so test data is always fresh. Production data doesn't exist yet (still in dev). Acceptable for now.

**W2 — `!!` force-unwrap on `myOrder` in `UserItemDetailContent`**
Line: `val display = if (rank != null) "Rank $rank / ${myOrder!!.size}" else "—"`
The `!!` is safe here because rank is only non-null when myOrder is non-null (rank is derived from myOrder?.indexOf(...)). The Kotlin compiler doesn't infer this from the chain, but it is logically impossible for myOrder to be null when rank is non-null. Risk: very low.

### MINOR

**M1 — `deleteConfirmation` is not preserved across tab switches**
`onTabSelected()` clears `selectedItem`, `selectedFeature`, and `evaluationEditor` but not `deleteConfirmation`. A pending deletion survives tab switches. This is actually OK UX — the dialog stays visible regardless of tab.
